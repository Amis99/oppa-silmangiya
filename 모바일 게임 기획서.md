# 모바일 게임 기획서  
## 오빠 실망이야 (가제)

---

## 1. 프로젝트 개요

- **게임명**: 오빠 실망이야 (가제)
- **플랫폼**: Android / iOS 공용 (크로스 플랫폼 앱)
- **장르**: 국어 학습 + 퍼즐형 캐주얼 게임
- **핵심 콘셉트**
  - 한국어 문장에서 **맞춤법, 표준어, 문장·글의 일관성·완결성·통일성**의 오류를 찾아 터치하는 게임
  - 각 스테이지마다 **세 개의 후보 구간 중 단 하나만 실제로 틀림**
  - 플레이할 때마다 **어느 구간이 틀릴지 랜덤으로 결정** → 외워서 풀기 어렵게 설계
  - 스테이지별로 **카카오톡 대화형 / 잡지·기사형 / 비문학형**이 자연스럽게 순차 등장
  - 초등학생도 시작 가능한 **레벨 1**부터, 국어 전문가 수준을 겨냥한 **레벨 100**까지

- **핵심 목표**
  - 재미 요소를 유지하면서도 **실제 국어 실력·한국어 감각 향상**에 도움이 되는 학습용 게임
  - 한국어 모국어 사용자뿐 아니라, **한국어를 공부하는 외국인 학습자**에게도 유용한 도구 제공

---

## 2. 타깃 유저

1. **국내 학습자**
   - 초등 고학년~중학생: 맞춤법·띄어쓰기 기본기 강화
   - 고등학생/수험생: 비문학 독해, 논리적 글읽기, 서술 일관성 감각 향상
   - 대학생/성인: 보고서·메일·문서 작성에서 정확한 국어를 쓰고 싶은 사용자

2. **해외 한국어 학습자**
   - **일본, 중국, 북미, 유럽, 중동·아프리카 등**에서 한국어를 배우는 학습자
   - K-POP, 드라마, 유학 준비, TOPIK 준비 등으로 한국어 학습 동기가 있는 사용자

3. **교사·강사·학원**
   - 수업 중 활용할 수 있는 **게임형 학습 자료**를 찾는 국어 교사·한국어 교원

---

## 3. 학습·제품 방향성

- **본문(문제 문장)은 100% 한국어**로 제공
- 메뉴·진행 안내·설정·랭킹 등 **UI 언어는 다국어 지원**
  - 예: 한국어 / 영어 / 일본어 / 중국어(간체·번체) / 아랍어 등
- 수익화는 **B2C 중심(앱 내 개별 사용자 대상)**으로 설계하고,  
  이후 필요 시 B2B(기관용) 확장 여지를 남겨 둠.

---

## 4. 게임 핵심 메커닉

### 4.1 기본 규칙

- 각 스테이지에는 **오류 후보 위치 3~10곳**이 존재
- 실제 플레이 시:
  - 이 중 **1곳만 ‘틀린 표현’**(오류)으로 활성화
  - 나머지 2곳은 **자동으로 올바른 표현**으로 바뀐 상태로 제시
- 유저는 글을 읽고 **어디가 틀렸는지 추론 후 해당 구간을 터치**
  - 정답: 해당 구간이 강조 표시되고, **올바른 표현 + 짧은 설명** 노출 + 햅틱 및 그래픽 효과
  - 오답: 라이프 1 감소 + 햅틱 및 그래픽 효과



### 4.2 시간 제한

- 스테이지마다 **제한 시간** 설정 (예: 40~90초 범위, 난이도별 조정)
- 제한 시간 내에 정답을 맞히지 못하면 해당 스테이지 **실패** + 햅틱 및 그래픽 효과


### 4.3 라이프 시스템

- 기본 라이프: **3개**
- 오답(오류가 아닌 부분 또는 빈 공간 터치) 시 라이프 1 감소
- 라이프 0이 되면 **스테이지 실패 / 게임 오버**
- 일정 포인트 달성, 연속 클리어 등의 조건 달성 시 **라이프 +1 보상** 제공

## 5. 모드(표현 형식) 구성

> 유저가 직접 모드를 고르는 것이 아니라, **레벨 설계에 따라 자연스럽게 섞여서 등장**하도록 구성합니다.

### 5.1 카카오톡 대화형 모드

- **UI 특성**
  - 카카오톡 스타일 채팅 레이아웃 (좌측/우측 말풍선, 아바타 아이콘)
  - 짧은 대화, 일상적인 표현 위주

- **학습 포인트**
  - 띄어쓰기, 맞춤법, 비표준어, 구어체에서의 표준어 사용
  - 예시 오류 유형
    - 안해도 → 안 해도  
    - 해야 되 → 해야 돼  
    - 맛없었잔아 → 맛없었잖아  
    - 벌써썼어요 → 벌써 썼어요 등

- **주요 사용 레벨**
  - 레벨 1~20 중심, 이후에도 간헐적으로 등장하여 난이도 환기 역할

### 5.2 잡지·기사형 모드

- **UI 특성**
  - 잡지 기사 카드 레이아웃
    - 제목, 부제, 기자명/출처, 2~4개 문단 구성
  - 학교 신문, 청소년 잡지, 홍보 기사 느낌

- **학습 포인트**
  - 기본 문장 구조, 명사+의존명사 띄어쓰기, 어색한 연결어, 경어법 등
  - 예시 오류 유형
    - 앉을데 → 앉을 데  
    - 이야기나누기에 → 이야기 나누기에  
    - 느껴보기도한다 → 느껴 보기도 한다 등

- **주요 사용 레벨**
  - 레벨 11~40 중심 (카톡과 비문학 사이의 중간 단계)

### 5.3 비문학형 모드

- **UI 특성**
  - 수능/모의고사 스타일의 짧은 비문학 지문 레이아웃
  - 정보 전달형, 설명문·논설문·탐구 보고서 등

- **학습 포인트**
  - 논리적 일관성(앞뒤 주장/설명 충돌), 통일성(용어·시점·관점 통일), 완결성(결론 누락/비약)
  - 문장 단위의 국어 규범 + 문단·글 단위의 논리 구조 판단

- **예시 오류 유형**
  - 같은 개념을 ‘자료’, ‘데이터’, ‘통계’ 등으로 혼용 사용
  - 앞에서 “A는 B이다”라고 정의해 놓고 뒤에서 “A는 B가 아니다”라고 서술
  - 지시어(이/그/저, 앞/뒤, 이전/다음)의 지칭 대상 불명확

- **주요 사용 레벨**
  - 레벨 31~100, 상위 레벨로 갈수록 비중 확대

## 6. 난이도 및 레벨 설계

### 6.1 레벨 구간 개요

1. **Lv. 1~10 (입문)**
   - 모드: 카톡 중심
   - 포인트: 기초 띄어쓰기, 고빈도 맞춤법, 쉽게 눈에 띄는 오류
   - 문장 길이 짧고, 문맥 단순

2. **Lv. 11~30 (초급)**
   - 모드: 카톡 + 잡지/기사 혼합
   - 포인트: 표준어/비표준어, 기본 호응, 어색한 표현

3. **Lv. 31~60 (중급)**
   - 모드: 잡지/기사 + 비문학
   - 포인트: 문단 내 시제·주어 일관성, 접속 표현, 요지/세부 내용 간 충돌

4. **Lv. 61~90 (상급)**
   - 모드: 비문학 중심
   - 포인트: 통일성, 논리적 정합성, 근거-결론의 연결

5. **Lv. 91~100 (마스터)**
   - 모드: 고난도 비문학 + 칼럼·논설형 텍스트
   - 포인트: 문서 전체의 품질(불필요한 반복, 모호한 표현, 논리 비약 등) 판별

### 6.2 모드 등장 패턴 (예시)

- Lv. 1~10: 카톡 80%, 잡지 20%  
- Lv. 11~30: 카톡 40%, 잡지 60%  
- Lv. 31~60: 잡지 40%, 비문학 60%  
- Lv. 61~100: 비문학 80%, 잡지 20%  
- 카톡 모드는 중간중간 “쉬어가는 레벨”로 산발적으로 재등장

## 7. 점수·라이프·랭킹 시스템

### 7.1 점수 시스템

- **정답 클릭**: +10점
- **스테이지 클리어 보너스**: +50점
- **남은 시간 보너스**: `남은 초 × 2점`
- 추후 확장:
  - 연속 클리어 보너스(예: 3연속 노데스 시 추가 포인트)

### 7.2 라이프 시스템

- 기본 라이프: 3개
- 오답 클릭 시 라이프 -1
- 라이프 0 → 해당 스테이지 실패, 결과 화면으로 전환
- 라이프 획득 방법:
  - 일정 누적 포인트 달성 시 라이프 +1
  - 연속 클리어 달성 보상
  - (무료 이용자 한정) 보상형 광고 시청 후 라이프 +1

### 7.3 랭킹 시스템

- **지표**
  - 총 누적 점수
  - 주간 획득 점수

- **구분**
  1. **전 세계(Global) 랭킹**
  2. **지역별 랭킹**
     - 아시아, 유럽, 북미, 남미, 중동·아프리카 등
  3. **국가별 랭킹** (유저 풀 축적 후 단계적으로 지원 가능)

- **표시 요소**
  - 순위, 닉네임, 레벨, 누적 점수, 국기/지역 아이콘
  - 나의 위치: “전 세계 120위 · 아시아 23위” 등 요약 표시

## 8. UX / 화면 흐름

### 8.1 오프닝 화면

- 게임 로고/타이틀: **오빠 실망이야**
- 한 줄 설명:
  - “누구에게나 사랑받는 똑똑한 사람이 되자!”
- 버튼(예시):
  - [새로 시작]  
  - [이어하기]
  - [랭킹]  
  - [설정]  

> 실제 구현 시에는 간단한 애니메이션(로고 등장, 버튼 튀어나오기 등)으로 연출

### 8.2 언어 선택 화면

- 첫 실행 시:
  - “UI 언어를 선택하세요”  
  - 한국어 / English / 日本語 / 简体中文 / العربية …  
- 선택 정보는 기기 로컬 + 서버 프로필에 저장
- 설정 메뉴에서 언제든 변경 가능

※ **게임 본문 텍스트는 항상 한국어**이나,  
HUD·안내·설정·랭킹 등은 선택한 언어로 표시

### 8.3 메인 화면

- 표시 요소
  - 현재 레벨 / 누적 점수 / 남은 라이프
  - 오늘의 추천 스테이지(향후 확장)
- 주요 버튼
  - [시작] → 다음 스테이지로 진행
  - [레벨 선택] (추후 확장)
  - [랭킹], [설정]

### 8.4 스테이지 화면

- 상단: HUD
  - 점수 / 남은 시간 / 라이프 / 레벨 표시
- 중앙: 모드별 콘텐츠 영역
  - 카톡: 채팅 말풍선
  - 잡지/기사: 기사 카드
  - 비문학: 지문 카드
- 하단: 진행 안내
  - “세 부분 중 하나만 틀려 있습니다. 잘못된 표현을 찾아 터치하세요.”  
  - 안내 문구는 선택된 UI 언어로 표시

### 8.5 결과 화면

- 성공 시
  - “스테이지 클리어!” (다국어)
  - 이번 스테이지에서 얻은 점수 + 남은 시간 보너스
  - 현재 누적 점수, 남은 라이프
  - 버튼: [다음 스테이지], [메인으로]

- 실패 시
  - “실패했습니다.” (다국어)
  - 실패 이유: 시간 초과 / 라이프 소진
  - 버튼:
    - 라이프 > 0: [다시 도전], [메인으로]
    - 라이프 = 0: [메인으로] (+ 보상형 광고로 라이프 1 회복 옵션)

## 9. 다국어 지원(i18n) 설계

### 9.1 기본 원칙

- **본문 텍스트**: 항상 한국어  
- **UI 텍스트**: 언어 리소스 파일로 관리, 선택 언어에 따라 교체
- 지원 언어(1차)
  - 한국어 (ko)
  - 영어 (en)
  - 일본어 (ja)
  - 중국어 간체 (zh-CN)
  - 중국어 번체 (zh-TW, 선택)
